The exam will consist of two parts.

  • You will be given both parts of the exam at the same time. You can work on both parts concurrently.

  • Understand the structure of an HTML5 document. Know what the DTD does.
  • Be able to discuss the goals of HTML5. Pave the cow paths. Don't break things, and so on.
  • Understand the concept of a document's nested (hierarchical) structure. Understand how elements must be "properly" nested.
  • You should be able to look at an HTML5 document and identify syntax errors and be able to correct them.
  • Understand the concept of elements and attributes. Know the rules for attributes (cannot have duplicate attribute names in an element, for example).
  • Be able to work with the elements that we have discussed in class. anchor, div, span, img, and table.
  • Understand the difference between block and inline tags.
  • Be able to discuss the purpose of the new "Semantic Tags", and how search engines might use these semantic tags. Understand the role of semantic tags in syndication and aggregation. We have talked about tags like article, section, nav, header, footer, etc..
  • Be able to work with tables and understand the structure of a table. Understand how a table is organized. (Tables have rows, which in turn has columns). Understand how to create jagged tables (tables where a cell spans multiple columns).
  • Understand the concept of white space handling and white space normalization.
  • Understand the concept of a URL and how it relates to the file system on the remote Web server.
  • Be able to work with cascading style sheets. Be able to work with the various types of selectors. Know how to create and use CSS classes and ID's. It goes without saying that you need to understand the role and rules of the ID attribute. Know how to embedded, internal, and external style sheets. Know the order in which rules are applied when one style overrides another.
  • Understand the concept of the CSS box model.
  • Understand the different ways to create a CSS syle using inline styles, embedded (internal) style sheets, and external style sheets.
  • Make sure that you understand the idea of inheritance and how properties (attributes) cascade.
  • Know the idea behind the various grids (grid 960). Know the idea behind Bootstrap. 
  • Be able to create CSS selectors.
  • Understand the purpose of Bootstrap and how to use it.  Don't try to remember all of the Bootstrap classes.

Practice Questions

  • HTML5 is living language. That is, the W3C standard is undergoing evolution. (true)

  • Semantic tags were introduced with some of the first versions of HTML. (false - Most of them appeared in HTML 5.)

  • The following is a valid HTML5 document: 
               <!DOCTYPE html>
					   <html>
						<head>
						<title>foo</title>
							</head>
						<body>
							</body>
					   </html>
    It is a valid document.

  • What is wrong with the following HTML5: 
                        <p class="Test" class="Test2">This is a line of text.</p>
    Two attributes cannot have the same name.

  • Which of the following correctly declares a CSS class named test?
    • #test{}
    • .cssclass test{}
    • .class test{}
    • .class="test"{}
    • .test{}

  • Create a table with three rows and two columns.
